## Overview

In [ACT 3 AI](/docs/legacy-wiki/act3ai "ACT3AI"), a **Set** is the environment or location where your [scenes](/docs/legacy-wiki/scenes "Scenes") and [shots](/docs/legacy-wiki/shots "Shots") take place.
Sets can be AI-generated, imported from 3D design tools, or customized in the [Editor workspace](/docs/legacy-wiki/editor "Editor") using the [Set Design](/docs/legacy-wiki/set-design "Set Design") system.
They provide the backdrop for [Digital Actors](/docs/legacy-wiki/digital-actors "Digital Actors"), props, and action, ensuring continuity across your project.

## Types of Sets

- **AI-Generated Sets:** Create immersive environments using text prompts (e.g., “Ancient temple at sunset”).
- **Template Sets:** Pre-built environments such as office, street, forest, sci-fi lab.
- **Imported Sets:** Import USD, FBX, or Blender assets for advanced worldbuilding.
- **Custom Sets:** Build from scratch using [Top-Down View](/docs/legacy-wiki/top-down-view "Top Down View") and props.

## How Sets Fit Into Workflow

- [AI Wizard](/docs/legacy-wiki/ai-wizard "AI Wizard") → Suggests set types based on story beats.
- [Script Editor](/docs/legacy-wiki/script-editor "Script Editor") → Associates sets with scene descriptions.
- [Set Design](/docs/legacy-wiki/set-design "Set Design") → Allows detailed customization.
- [Top-Down View](/docs/legacy-wiki/top-down-view "Top Down View") → Place cameras, props, and actors.
- [Storyboards & Panels](/docs/legacy-wiki/storyboard-panels "Storyboard Panels") → Visualize how sets appear in shots.
- [Rendering System](/docs/legacy-wiki/rendering-system "Rendering System") → Generates the final set visuals for previews and renders.

## Working with Sets

1. From the [Editor workspace](/docs/legacy-wiki/editor "Editor"), go to **Set Management**.
2. Create or load a set.
3. Add props, furniture, and background details.
4. Place [actors](/docs/legacy-wiki/digital-actors "Digital Actors"), [lighting](/docs/legacy-wiki/lighting "Lighting"), and [cameras](/docs/legacy-wiki/cameras "Cameras").
5. Save the set to the [Asset Library](/docs/legacy-wiki/asset-library "Asset library") for reuse across projects.

## Example Use Cases

- A consistent office environment across multiple corporate [marketing videos](/docs/legacy-wiki/marketing-videos "Marketing Videos").
- Fantasy landscapes for a film or animation project.
- Reusable streets, parks, or interiors for episodic series.
- Minimalist studio sets for product explainers.

## Best Practices

- Use consistent sets across related scenes for continuity.
- Save sets to the [Asset Library](/docs/legacy-wiki/asset-library-2 "Asset Library") for quick reuse.
- Test camera paths in [Top-Down View](/docs/legacy-wiki/top-down-view "Top Down View") before final renders.
- Apply [lighting presets](/docs/legacy-wiki/lighting "Lighting") to unify tone across different sets.

## Troubleshooting

- **Set not loading** → Recheck asset compatibility or clear cache.
- **Actors not visible in set** → Verify placement in [Top-Down View](/docs/legacy-wiki/top-down-view "Top Down View").
- **Render too complex** → Simplify set elements or reduce resolution in [Render Queue](/docs/legacy-wiki/render-queue "Render Queue").
- **Inconsistent visuals** → Apply a global [LUT or color grading](/docs/legacy-wiki/color-grading "Color Grading") profile.
