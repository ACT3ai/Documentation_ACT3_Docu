## Overview

The **Documentation** hub centralizes all product guides, specs, and how‑tos for [ACT 3 AI](/docs/legacy-wiki/act3ai "ACT3AI").
Use it to navigate onboarding, creative workflows, integrations, billing, and admin topics. Everything here links to detailed sub‑pages so your team can move from idea to export with confidence.

## Who This Is For

- New users starting with [Getting Started](/docs/legacy-wiki/getting-started "Getting Started")
- Creators and editors working in the [Editor](/docs/legacy-wiki/editor "Editor")
- Technical users integrating models like [Google Veo 3](/docs/legacy-wiki/google-veo-3 "Google Veo 3"), [Runway](/docs/legacy-wiki/runway "Runway"), and [WAN AI](/docs/legacy-wiki/wan "Wan")
- Org admins managing [billing](/docs/legacy-wiki/billing-model "Billing Model") and [credits](/docs/legacy-wiki/credit-system "Credit System")

## How to Use This Hub

1. Start with [Getting Started](/docs/legacy-wiki/getting-started "Getting Started") → create a [New Project](/docs/legacy-wiki/new-project "New Project").
2. Shape your story with [AI Wizard](/docs/legacy-wiki/ai-wizard "AI Wizard") (acts, beats, logline).
3. Build assets and performances with [Characters](/docs/legacy-wiki/characters "Characters") and [Digital Actors](/docs/legacy-wiki/digital-actors "Digital Actors").
4. Import or write scripts via [Import Script](/docs/legacy-wiki/import-script "Import Script") and edit in the [Script Editor](/docs/legacy-wiki/script-editor "Script Editor").
5. Plan & block scenes in [Storyboard & Panels](/docs/legacy-wiki/storyboard-panels "Storyboard Panels") and [Top‑Down View](/docs/legacy-wiki/top-down-view "Top Down View").
6. Generate previews/renders using [Google\_Veo\_3](/docs/legacy-wiki/google-veo-3 "Google Veo 3"), [Runway](/docs/legacy-wiki/runway "Runway"), or [WAN AI](/docs/legacy-wiki/wan "Wan").
7. Assemble and export in [Build Video](/docs/legacy-wiki/build-video "Build Video"); track jobs in [Render Queue](/docs/legacy-wiki/render-queue "Render Queue").
8. Manage costs with [Billing Model](/docs/legacy-wiki/billing-model "Billing Model") and [Credit System](/docs/legacy-wiki/credit-system "Credit System").

## Style & Conventions

- Use sentence‑case headings and short paragraphs.
- Link the first mention of features/pages (e.g., [Editor](/docs/legacy-wiki/editor "Editor"), [Render\_Queue](/docs/legacy-wiki/render-queue "Render Queue")).
- Prefer ordered steps for procedures; bullets for options.
- Keep examples concise; add screenshots where available.

## Templates

- **Page header:** one‑sentence purpose + who it’s for.
- **Sections:** Overview → Key Capabilities → How to Use → Tips → See Also.
- Include a **Troubleshooting** section for task pages (e.g., [Import\_Script](/docs/legacy-wiki/import-script "Import Script"), [Build\_Video](/docs/legacy-wiki/build-video "Build Video")).

## Versioning & Changelog
