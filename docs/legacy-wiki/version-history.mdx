## Overview

The **Version History** feature in [ACT 3 AI](/docs/legacy-wiki/act3ai "ACT3AI") provides a detailed record of all changes made to [projects](/docs/legacy-wiki/projects "Projects"), [scripts](/docs/legacy-wiki/scripts "Scripts"), [scenes](/docs/legacy-wiki/scenes "Scenes"), and [video assemblies](/docs/legacy-wiki/build-video "Build Video").
It tracks edits, saves, and renders over time, allowing creators and teams to review, compare, and restore earlier iterations.

## Key Features

- **Change Log:** Full timeline of edits with timestamps and contributor names.
- **Snapshots:** Auto-saved points created during [script editing](/docs/legacy-wiki/script-editing "Script Editing"), [timeline adjustments](/docs/legacy-wiki/timeline-view "Timeline View"), and [video editing](/docs/legacy-wiki/build-video "Build Video").
- **Manual Checkpoints:** Users can mark important milestones (e.g., “Draft v1,” “Final Cut”).
- **Comparison Tools:** Highlight differences between two versions of a script, timeline, or storyboard.
- **Rollback & Restore:** Recover older versions without losing new ones.

## Where Version History Applies

- **Scripts:** Scene revisions, AI-generated rewrites, and dialogue changes.
- **Timeline:** Adjustments in pacing, sequencing, and transitions.
- **Storyboards:** Visual changes in [Storyboard Panels](/docs/legacy-wiki/storyboard-panels "Storyboard Panels").
- **Digital Assets:** [Sets](/docs/legacy-wiki/set-design "Set Design"), [Digital Actors](/docs/legacy-wiki/digital-actors "Digital Actors"), and [style presets](/docs/legacy-wiki/style-presets "Style Presets").
- **Final Renders:** Records of outputs in [Render Queue](/docs/legacy-wiki/render-queue "Render Queue") and [exports](/docs/legacy-wiki/export-distribution "Export Distribution").

## How to Access

1. Open your [Editor Workspace](/docs/legacy-wiki/editor "Editor") or [Project Dashboard](/docs/legacy-wiki/projects "Projects").
2. Click **File → Version History**.
3. Browse the chronological list of changes.
4. Select one or more versions to preview.
5. Choose to duplicate, rollback, or lock.

## Collaboration Integration

- Works with [Organization Management](/docs/legacy-wiki/organization-management "Organization Management") to track changes by user.
- Approved versions can be locked to prevent further changes.
- Reviewers can comment on specific versions without altering them.

## Best Practices

- Use manual checkpoints for critical milestones.
- Name versions clearly (e.g., “Client Feedback v2”).
- Regularly review history to avoid unnecessary [credit consumption](/docs/legacy-wiki/credit-system "Credit System").
- Lock final versions for compliance and delivery.

## Troubleshooting

- **History not updating** → Ensure auto-save is enabled.
- **Too many versions** → Archive older versions or branch projects.
- **Rollback failed** → Re-open the [Editor Workspace](/docs/legacy-wiki/editor "Editor") and reapply.
- **Missing contributor data** → Check [team roles and permissions](/docs/legacy-wiki/organization-management "Organization Management").

## Example Workflow

1. Script created via [AI Wizard](/docs/legacy-wiki/ai-wizard "AI Wizard").
2. Revised in [Script Editor](/docs/legacy-wiki/script-editing "Script Editing").
3. Saved as **v1.0 – First Draft**.
4. Storyboards added in [Storyboard Panels](/docs/legacy-wiki/storyboard-panels "Storyboard Panels").
5. Final locked as **v2.0 – Approved** before sending to [Render Queue](/docs/legacy-wiki/render-queue "Render Queue").
